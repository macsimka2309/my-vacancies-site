Цель MVP — запустить сайт с вакансиями, принимать отклики кандидатов, отправлять их в Telegram ответственному менеджеру и сохранять все лиды в базе/таблице для дальнейшей обработки и аналитики.


В MVP входит:
1. Публичный сайт с вакансиями.
2. Страница каждой вакансии.
3. Форма отклика.
4. Отправка отклика в Telegram-канал/чат.
5. Сохранение отклика в базу данных.
6. Простая админка или таблица для просмотра лидов.
7. Статусы обработки кандидатов.


В первый MVP не входит:
1. Авторизация кандидатов.
2. Личный кабинет кандидата.
3. ИИ-интервью.
4. Автоматический скоринг кандидатов.
5. Сложная CRM.
6. Интеграции с hh.ru, Avito, SuperJob.
7. Массовые рассылки.
8. Сложная аналитика.


Главный процесс MVP
1. Кандидат заходит на сайт.
2. Видит список вакансий.
3. Открывает интересующую вакансию.
4. Читает условия.
5. Нажимает "Откликнуться".
6. Заполняет форму.
7. Нажимает "Отправить".
8. Система сохраняет отклик.
9. Система отправляет уведомление в Telegram.
10. Менеджер видит отклик.
11. Менеджер связывается с кандидатом.
12. Менеджер обновляет статус лида



Структура вакансии

Вакансия:
- ID вакансии
- Название
- Компания / проект
- Город
- Формат работы
- Зарплата
- График

- Требования
- Условия
- Адрес / район
- Контактный комментарий
- Активна / неактивна
- Дата создания
- Дата обновления
